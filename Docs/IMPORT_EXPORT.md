# Импорт и экспорт каталога (YAML)

## 1. Назначение

Прайс магазина описывается файлом **YAML** единого формата (как в учебном репозитории Netology). Импорт создаёт или обновляет записи **`Shop`**, **`Category`**, **`Product`**, **`ProductInfo`**, **`Parameter`**, **`ProductParameter`**. Экспорт строит такой же YAML из текущего состояния БД для магазина.

## 2. Структура файла

Корневой объект:

| Ключ | Тип | Описание |
|------|-----|----------|
| `shop` | строка | Название магазина (создание/привязка `Shop`). |
| `categories` | массив | Элементы: `id` (число), `name` (строка). Id из файла сохраняются в БД как первичные ключи категорий. |
| `goods` | массив | Список товарных позиций. |

Элемент **`goods`**:

| Поле | Описание |
|------|----------|
| `id` | Внешний идентификатор → `ProductInfo.external_id`. |
| `category` | Id категории из массива `categories`. |
| `model` | Строка модели. |
| `name` | Название продукта. |
| `price`, `price_rrc`, `quantity` | Цена, РРЦ, остаток. |
| `parameters` | Объект «имя параметра → значение»; значения приводятся к строке при сохранении. |

## 3. Поведение импорта

Реализация: **`catalog/services/yaml_catalog.py`**, функция **`import_price_from_mapping`**.

1. Магазин **`get_or_create`** по имени; пользователь-владелец задаётся **`shop_user_id`**.  
2. Категории: **`get_or_create`** по **`id`** из файла; связь категории с магазином через M2M **`shops.add`**.  
3. Для данного магазина удаляются все **`ProductInfo`** (и каскадно параметры), затем создаются заново из **`goods`**.  
4. Продукты: **`get_or_create`** по паре **`name`** + **`category_id`**.

Транзакция: вся операция в **`atomic`**.

## 4. Способы запуска импорта

### 4.1. Команда Django (локальный файл)

```bash
python manage.py import_shop_yaml путь/к/файлу.yaml --email user@shop.com
```

Пользователь с указанным email должен существовать и иметь **`type=shop`**.

Пример из репозитория:

```bash
python manage.py import_shop_yaml data/shop1.yaml --email partner@example.com
```

### 4.2. API партнёра

**`POST /api/v1/partner/update`** с телом **`{"url": "https://..."}`** — загрузка по HTTP (таймаут запроса ограничен в коде). Требуется токен пользователя **`shop`**.

### 4.3. Celery (администратор)

**`POST /api/v1/admin/import_task`** с **`url`** и **`shop_user_id`** — постановка задачи **`do_import_task`**. Нужны права **`is_staff`** и запущенные Redis + worker.

## 5. Экспорт

**`GET /api/v1/partner/export`** — пользователь **`shop`**, ответ: YAML с полями `shop`, `categories`, `goods` и блоком **`parameters`** у каждой позиции.

Реализация: **`catalog/services/yaml_export.py`**, функция **`dump_shop_catalog_yaml`**.

## 6. Типичные ошибки

- **403 «Только для магазинов»** — у пользователя не выставлен тип **`shop`**.  
- **Магазин не найден** при экспорте — для пользователя ещё не создан объект **`Shop`** (сначала выполните импорт прайса или создайте магазин в админке).  
- **Некорректный YAML** — проверьте отступы и типы полей по примеру **`data/shop1.yaml`**.
